18. ExpAT HTML Renderer Tests

This chapter tests the expat-based HTML renderer used to display dictionary definitions. There are seven tests, one per lookup word. Each test describes what you should see on the definition screen.

Dictionary

html-definitions

Note

Do not use html-definitions for any other chapter in this book.

Test A: BlazeSilent (Abbr title expansion)

Look up BlazeSilent. The definition contains three abbr elements, each with a title attribute. The renderer appends the title in parentheses after the abbreviation text.

Expected output (three lines after the description):

  • Abbreviation one: c. (circa)
  • Abbreviation two: AD (anno Domini)
  • Abbreviation three: f. (filius) -- the abbreviation text is italic

Test B: ClearSvg (Block-strip tags)

Look up ClearSvg. The definition contains nine block-level tags whose entire subtrees must be stripped: hiero, svg, math, gallery, nowiki, poem, ref (lowercase), REF (uppercase), and a standalone img.

Expected output: the description paragraph, then a ---------- separator, then nothing, then a second ---------- separator. If any text appears between the two rules, a strip tag is leaking content.

Test C: DarkMath (Block structure)

Look up DarkMath. The definition exercises every block structure element.

Expected output (in order, after the description):

  • Two separate paragraphs (p)
  • Two div blocks, each on its own line (div)
  • Three lines separated by line breaks (br)
  • An indented passage: "indented passage" (blockquote)
  • Numbered list: first, second, third (ol + li)
  • Bulleted list: alpha, beta, gamma (ul + li)
  • Four bold headings: Heading One, Heading Two, Heading Three, Heading Four (h1--h4)

Test D: EmptyGallery (Inline formatting)

Look up EmptyGallery. The definition exercises every inline formatting tag. Expected rendering for each word or phrase:

  • bold (b element) -- rendered bold
  • bold (strong element) -- rendered bold
  • italic (i element) -- rendered italic
  • italic (em element) -- rendered italic
  • underline (u element) -- rendered underlined
  • strike (s element) -- rendered as plain text (strikethrough not supported by renderer)
  • H2O (sub element) -- rendered as plain text "H2O" (subscript not supported)
  • x2 (sup element) -- rendered as plain text "x2" (superscript not supported)
  • printf (code element) -- rendered bold
  • mono (tt element) -- rendered bold
  • small (small element) -- rendered as plain text (font-size change not supported)
  • big (big element) -- rendered as plain text (font-size change not supported)
  • count (var element) -- rendered italic

Nested combinations (on a second line):

  • bold-italic (b + i) -- rendered bold-italic
  • bold-italic-underline (b + i + u) -- rendered bold-italic-underlined

Spacing check: the words on the inline formatting line are separated by spaces in the source. Verify that a visible gap appears between every consecutive pair of words on screen -- no two styled words should be touching or run together.

Test E: FrostNowiki (Strip tag, keep text)

Look up FrostNowiki. The definition contains four cases where a tag is stripped but its text content is preserved.

Expected output (four lines after the description):

  • visible span text (span tag stripped, text kept)
  • visible unknown text (unknown tag stripped, text kept)
  • visible nested text (nested unknown tags both stripped, innermost text kept)
  • visible anchor text (anchor tag stripped, text kept)

Test F: GlowPoem (Wikitext annotations)

Look up GlowPoem. The definition contains eight self-closing wikitext annotation tags of the form XX:YY. The renderer emits the suffix (the part after the colon) as inline text; the tag itself does not appear. A ninth case uses a tag with body text; the body is suppressed and only the suffix renders.

Expected inline text in order: four, oikos, la, sharp, noun, Grek, ameba, female, value

Test G: HazeEntity (HTML named entity resolution)

Look up HazeEntity. The definition contains five HTML named entities. The renderer resolves them to their UTF-8 equivalents inline before passing the text to the XML parser.

Expected output (five lines after the description):

  • Brackets: [enclosed] (lsqb / rsqb resolved to [ ])
  • Non-breaking space: left right (nbsp resolved to space)
  • En dash: 1939-1945 (ndash resolved to UTF-8 en dash)
  • Zero-width: before (lrm silently dropped)
  • Unknown: before (unknown entity silently dropped)
